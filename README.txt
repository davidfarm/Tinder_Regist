Tinder Register



В этом архиве ты можешь найти следующие файлы:
	1. config.ini
	2. config_browser.txt
	3. Tinder_Register_vX.Y.Z.K

config.ini предназначен для установки в скрипт следующих параметров:
	1. photos_dir - папка с фотографиями на твоём компьютере
	2. geo - указывает на то, в каком регионе будет регестрироваться аккаунт(доступные гео прописаны в config.ini)
	3. proxy_country, proxy_city - обычно не используются
	4. group_id - название твоей папки в мультилогине в которую будут регестрироваться аккаунты
	5. reg_variable - пол человека на которого будет регестрироваться аккаунт
	6. change_account_settings - нужно ли в конце работы скрипта добавлять к нему настройки возраста и т.п.
Важно! внимательно смотреть за конфигом, чтобы случайно не зарегать 100500 аккаунтов не в ту папку, не на тот регион или вообще женский профиль на мужской аккаунт

config_browser.txt предназначен для тех, кто хочет создать новую папку в который будут регестрироваться аккаунты, если твоя папка стандартна (FA001-FA009) - можешь смело удалять файл.Данные тут представлены в формате JSON, т.е. сначала идёт "название папки в браузере" а после знака : идёт { "порт для мультилогина": "group айди твоей папки в мультилогине", "порт для индиго": "group айди твоей папки в индиго" }
group айди твоей папки берётся из программы POSTMAN.Сначала создай нужную тебе папку в нужном тебе браузере, после чего помести в неё любой профиль.В POSTMAN тебе нужно послать get-запрос по адресу http://localhost:port/api/v2/profile/ где вместо port укажи своё значение порта для браузера.После получения ответа, нажми на него, после чего используя сочитание клавиш ctrl+f найди название своего профиля, который ты ранее поместил в нужную для тебя папку.Ты увидешь примерно следующее:
{
        "uuid": "e72c36d3-4ed9-46f3-b2ca-649bfd546977",
        "group": "2dd80ced-4ea5-457d-a233-8286dfc5d723",
        "name": "AM27/MV27//24.05./ (GEO Perevod) ⌛ block",
        "notes": "",
        "browser": "mimic",
        "browserNeedsUpdate": null,
        "appNeedsUpdate": null,
        "updated": 1687010641.000000000
    }
тебе нужно значение "group".
Важно! Если ты добавляешь новую папку в конфиг для браузера то нужно внимательно смотреть на запятые.Перед каждой новой указанной папкой должна стоять запятая.В конце списка папок запятую ставить не нужно.Пример с предустановленными значениями в config_browser.txt.

В архиве нет папки res, если у тебя нет её на компьютере - создай.В ней должны быть следующие файлы:
	1. gmail.xlsx - в первой колонке будут адреса эл.почт, во второй пароли к ним, в третьей запасные почты, иногда требуются для подтверждения
	2. session_names - файл в котором в каждой новой строке будет вписан FAST( скомпилированное название для сессии, которое берётся из гугл таблицы )

После первого запуска скрипта у тебя появятся ещё две папки, это log и invalid sessions.В логах содержится информация о том, как отработал скрипт, в invalid sessions - названия незарегестрированных сессий.
	


Теперь о том, как работает скрипт.После запуска он сначала проверяет прокси.После чего создаёт пустую сесию в браузере.Потом он авторезируется в гугле и тиндере.В эот момент наиболее часто встречаются ошибки когда скрипт не справляется с капчей или не приходит смс-ка.После этого он регестрирует сам аккаунт и завершает работу переименованием сессии( в конец названия дописывает !reg)
При возникновении ошибок у тебя есть команды для управления, чтобы подробнее про них узнать - пиши info при возникновении ошибки.

Важно ! Скрипт постоянно дорабатывается, и некоторая инфа отсюда может быть неактуальна
